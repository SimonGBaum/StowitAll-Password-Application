# StowitAll Application
## Fundamental Questions
### What is my application?
-   A one-stop shop for all your password needs where you can:
    - Create new passwords
    - Store passwords
    - Link passwords to their appropriate site
    - Ensure passwords meet security standards
### What Problem does this application Solve?
- This application helps the customer:
    - Create encrypted and secure passwords
    - Store passwords so they are not forgotten
    - Links the passwords to the sites and features they were created for
- This helps create organization and flexibility:
    - Organize their passwords
    - Can make their own passwords
    - Can let the password-smith create a password
- This application helps resolve:
    - Trying to remember the hundreds of passwords, that are supposed to all be unique and difficult to hack.
### Who Are the Users?
- Who will use the application?
    - Anyone; no limitations on who could find value in this application
- Are there multiple user Types?
    - No
- What are their responsibilities?
    - Provide personal information
    - Personal Information:
        - email
        - name
    - Password Information:
        - site it is for
        - own password
        - what type of password they want from the password-smith
- What information does each user need?
    - Access to their profile
    - User Profile will have:
    - Frist Name
    - Last Name
    - Email
    - Username
    - Password
### What Are the User Goals? (Clarify Your Features)
- What are users trying to accomplish?
    - To create secure and unique password
    - To be able to store their passwords
    - To link them to the site and features that they are for
- What are the top 3 reasons they log in?
    1.  all their passwords are in one location
    2.  can easily find password details
    3.  can edit their passwords quickly and effortlessly
- What tasks do they perform most frequently?
    - Create a new password
    - Look up a password
    - Edit an existing password
    - Delete an old password
### What Information Must Users See?
- What information should users view?
    - Their profile
    - Their passwords
- What information should users create?
    - Their profile 
    - Their passwords (if they want to)
- What information should users update?
    - Their profile
    - Their passwords
- What information should users delete?
    - Their passwords
- What information should users never see?
    - Other users’ profiles
    - Other users’  passwords
### What Actions Can Users Perform?
- Create Something?
    - Yes
    - Profile
    - passwords
- Read Something?
    - Yes
    - Profile information
    - Password information
- Update Something?
    - Yes
    - Profile information
    - Password information
- Delete something?
    - Yes
    - Password
- Search?
    - Yes
    - Passwords
- Sort?
    - No
- Filter?
    - No
- Export?
    - No
- Share?
    - No
### What Pages Are Required?
    1.  Login / Sign-Up
    2.  Home
    3.  Contact Us
    4.  Error Message
    5.  Profile
    6.  Create
    7.  Vault
### What Data Must Be Collected?
- Users' data:
    - User first name
    - User last name
    - User email
### What Permissions Exist?
- Member/User:
    - Access by any person signed in, authenticated and authorized
### What Happens When There Is an Error?
- Login fails:
    -  Show a clear error:
    - "Incorrect username or email"
- No internet connection:
   -   Display offline banner
   -   Queue actions for retry when reconnected
- All other errors:
   -   Redirect to the Error Page
